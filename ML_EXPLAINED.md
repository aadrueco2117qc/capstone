# EyeGuard — How the Machine Learning Works

> This file is automatically updated when ML files are edited.

---

## Overview

The ML system takes a user's daily health log and outputs three things:
- A **risk level** (Low / Moderate / High / Severe)
- A **risk percentage** (0–100%)
- A **fatigue score** (0–10)

It uses three separate scikit-learn models, each trained on 9,000 synthetic data points.

---

## The Full Pipeline (Step by Step)

```
User submits daily log
        ↓
[ Feature Engineering ]   → converts raw log into 12 numbers
        ↓
[ StandardScaler ]        → normalizes all 12 numbers (mean=0, std=1)
        ↓
        ├──→ [ Random Forest Classifier ]      → risk level (0–3)
        ├──→ [ Gradient Boosting Regressor ]   → risk percentage (0–100)
        └──→ [ Linear Regression ]             → fatigue score (0–10)
        ↓
[ Recommendations ]       → picked based on risk level
        ↓
Result returned to dashboard
```

## Step 1 — Training Data

**File:** `backend/ml/training.py` → `SyntheticDataGenerator`

The models do NOT train on real user data. They train on **9,000 fake-but-realistic logs**
generated by `SyntheticDataGenerator.generate_synthetic_logs()`.

When training runs, the pipeline prints a detailed progress log to the console broken into
four labeled steps: data generation (with risk-level distribution and sample averages),
feature engineering (listing all 12 features), model training, and a final summary of
cross-validation scores. This output is cosmetic only — it does not affect model behavior.

It simulates **300 fake users × 30 days** each. For every day it randomly generates:

| Field | Range |
|---|---|
| Screen time | 4–14 hours (with ±1hr daily noise) |
| Break minutes | 0 to `(screen_time / 12) * 60` |
| Symptoms | 0–3 symptoms, sampled from a normalized distribution (see below) |
| Sleep quality | 3–10 |
| Water intake | 2–10 cups |
| Break type | outdoor, rest, exercise, meditation, walk, gaming, social_media |
| Eye exercises | 0–4 |
| Blue light filter | True / False |

The **symptom count** for each sample is sampled from `[0, 1, 2, 3]` using a probability
distribution derived from screen time and break minutes:

```
symptom_prob = clip((screen_time / 16) × (1 − break_minutes / 60), 0, 1)
p0 = max(0, 1 − symptom_prob)   # probability of 0 symptoms
p1 = symptom_prob × 0.5         # probability of 1 symptom
p2 = symptom_prob × 0.3         # probability of 2 symptoms
p3 = symptom_prob × 0.2         # probability of 3 symptoms
# All four are normalized to sum to 1.0 before sampling
```

The **risk label** for each sample is calculated as:
```
risk_percentage = (screen_time / 16) * 100 + (num_symptoms * 15) - (break_minutes * 0.5)
risk_level      = base_pattern + (1 if num_symptoms > 2 else 0)   [capped at 3]
```

The **fatigue label** is calculated as:
```
fatigue = (risk_level * 2.5) + (screen_time / 16 * 3)
        - (break_effectiveness * 1.5)
        - (sleep_quality / 10 * 2)
        - (water_intake / 8 * 1)
        + random noise (±0.5)
```

## Step 2 — Feature Engineering

**File:** `backend/ml/features.py` → `FeatureEngineer.extract_features()`

Raw log data is converted into a **12-number vector** before going into any model.

| # | Feature Name | How It's Calculated |
|---|---|---|
| 0 | `screen_time_minutes` | `screen_time_hours × 60` |
| 1 | `break_minutes` | direct from log |
| 2 | `symptom_index` | weighted average of symptom severities (see table below), capped at 1.0 |
| 3 | `sleep_quality` | `sleep_quality / 10` |
| 4 | `water_intake_ratio` | `cups / 8`, capped at 1.0 |
| 5 | `break_effectiveness` | break type score × duration multiplier (see table below) |
| 6 | `break_ratio` | actual break time ÷ recommended (5 min per hour of screen time) |
| 7 | `symptom_count` | `len(symptoms)` |
| 8 | `had_outdoor_break` | `1.0` if break_type contains "outdoor", else `0.0` |
| 9 | `eye_exercises_count` | direct from log |
| 10 | `blue_light_filter_enabled` | `1.0` if True, else `0.0` |
| 11 | `screen_time_ratio` | `screen_minutes / (24 × 60)` — % of the full day on screen |

### Symptom Severity Weights

| Symptom | Weight |
|---|---|
| mild_eye_discomfort | 0.30 |
| moderate_eye_strain | 0.50 |
| dry_eyes | 0.55 |
| headache | 0.60 |
| difficulty_focusing | 0.65 |
| blurred_vision | 0.70 |
| eye_fatigue | 0.70 |
| severe_eye_strain | 0.80 |

### Break Type Effectiveness Scores

| Break Type | Score |
|---|---|
| gaming | 0.20 |
| social_media | 0.30 |
| rest | 0.60 |
| meditation | 0.75 |
| exercise | 0.80 |
| walk | 0.85 |
| outdoor | 0.90 |

Break duration also affects effectiveness:
- Under 5 min → score scaled down proportionally
- 5–15 min → full score (optimal)
- 15–30 min → 90% of score
- Over 30 min → 70% of score

## Step 3 — The Three Models

**File:** `backend/ml/models.py`

All three models receive the same 12-feature vector (after StandardScaler normalization).

### Model 1 — Risk Level Classifier
- **Algorithm:** Random Forest
- **Config:** 200 trees, max depth 15, min samples split 5
- **Output:** Integer 0–3 → `Low / Moderate / High / Severe`
- **Also outputs:** Probability for each class (e.g. 60% Moderate, 30% High)
- **Validation:** ~85% accuracy (5-fold cross-validation on synthetic data)

How Random Forest works here: 200 decision trees each independently vote on the risk level.
The majority vote wins. The probability is the fraction of trees that voted for each class.

### Model 2 — Risk Percentage Regressor
- **Algorithm:** Gradient Boosting
- **Config:** 200 estimators, learning rate 0.1, max depth 8, subsample 0.8
- **Output:** Float 0–100 (clamped to that range)
- **Validation:** R² = 0.82 (5-fold cross-validation)

How Gradient Boosting works here: 200 trees built one at a time. Each new tree focuses on
correcting the errors of all previous trees. The final prediction is the sum of all trees.

### Model 3 — Fatigue Predictor
- **Algorithm:** Linear Regression
- **Output:** Float 0–10 (clamped to that range)
- **Validation:** R² = 0.75 (5-fold cross-validation)

How Linear Regression works here: a simple weighted sum of the 12 features plus an intercept.
`fatigue = w0*feature0 + w1*feature1 + ... + w11*feature11 + intercept`

## Step 4 — Saving & Loading Models

**File:** `backend/ml/storage.py` → `ModelStorage`, `PredictorManager`

After training, four objects are saved as `.pkl` files using Python's `pickle`:

```
backend/ml/models/
  risk_classifier.pkl       ← Random Forest + its StandardScaler
  risk_regressor.pkl        ← Gradient Boosting + its StandardScaler
  fatigue_predictor.pkl     ← Linear Regression + its StandardScaler
  feature_engineer.pkl      ← FeatureEngineer (with fitted scaler state)
```

On Flask startup (`app.py`), `PredictorManager.get_predictor()` runs:
1. Check if all 4 `.pkl` files exist
2. If yes → load them (fast, ~1 second)
3. If no → run full training pipeline (~30 seconds), then save

## Step 5 — Making Predictions

**File:** `backend/ml/predictor.py` → `RiskPredictor`

### Today's Prediction (`predict_today`)
Takes one daily log → runs through all three models → returns combined result.

### Tomorrow's Prediction (`predict_tomorrow`)
Averages the last 7 days of logs into one feature vector, then bumps screen time up by 5%
to simulate a slightly worse next day.

### 7-Day Forecast (`predict_week`)
Runs `predict_today` on each of the last 7 logs, assigning dates in order.

### Trend Detection (`_calculate_trend`)
Compares average risk of the **last 3 predictions** vs. **all older predictions**:
- Difference > +0.5 → `"worsening"`
- Difference < -0.5 → `"improving"`
- Otherwise → `"stable"`

### Recommendations
Hardcoded lists per risk level, selected after the classifier outputs a level:

| Risk Level | Example Recommendations |
|---|---|
| Low (0) | "Great job maintaining healthy screen habits!", "Continue taking regular breaks every hour." |
| Moderate (1) | "Consider increasing break frequency.", "Try doing eye exercises every 2 hours." |
| High (2) | "Take a 15-minute break immediately.", "Use the 20-20-20 rule." |
| Severe (3) | "URGENT: Take a 30-minute break from screens.", "Schedule an eye care appointment." |

---

## Key Limitation

The models were trained entirely on **synthetic data**, not real student responses.
The accuracy numbers reflect performance on that fake data.
Once real users log consistently, retraining on actual data would make predictions
genuinely personalized and more accurate.
